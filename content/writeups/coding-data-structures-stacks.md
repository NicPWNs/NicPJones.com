<p>Let’s cover a basic coding data structure: <em>stacks</em>. Stacks are a basic data structure that are like lists and arrays, but have a last in, first out (LIFO) property when adding and removing elements. Elements in a stack can either be “pushed” onto the stack to be added, or “popped” from the stack to be removed. A classic example is a heavy stack of dishes where plates can only be added and removed from the top.</p><figure><img alt="Stacks are like a stack of plates." src="/writeups/coding-data-structures-stacks/img-1.jpg"  decoding="async" width="330" height="446" loading="eager" fetchpriority="high" /><figcaption>Stacks are like a stack of plates.</figcaption></figure><h2>Stack Big-O Complexity</h2><figure><img alt="Big-O Complexity" src="/writeups/coding-data-structures-stacks/img-2.png"  decoding="async" width="997" height="716" loading="lazy" /><figcaption>Big-O Complexity</figcaption></figure><h3>Time Complexity</h3><p><strong>Access/Search:</strong> O(n) — Linear time. Stacks must be looped/iterated through to locate elements.</p><p><strong>Insertion/Deletion:</strong> O(1) — Constant time. Elements can be added or removed by pushing onto the stack or popping from the stack.</p><h3>Space Complexity</h3><p><strong>All Cases: </strong>O(n) — Linear space. Stacks contain a linear amount of data. As the stack increases in size, so does the space used.</p><h2>Stack Use Cases</h2><ul><li>Program Function Calling — Programming interpreters and compilers keep track of function call execution in a stack. If function a() calls function b(), function a() will be pushed onto the stack with function b() on top of it. When function b() finishes execution to return to function a(), it will be popped from the stack.</li><li>Undo (CTRL+Z) — Undo operations in applications like Microsoft Word use a stack to undo or “pop” the last edit made from the stack.</li></ul><h2>Stack Python Implementation</h2><p>The regular Python list data type has pop() and append() functions that can make it work as a stack. However, Python lists are dynamic arrays and fully reassign memory locations whenever they grow past their originally assigned size in memory. This can make Python lists very inefficient to use for stacks, so the recommended approach is to use deque from the Python standard collections library which resolves this issue.</p><p>My full stack Python implementation can be found on my <a href="https://github.com/NicPWNs/CodingPractice/blob/main/data-structures-algorithms/data-structures/stack/stack.py">GitHub</a>. I’ve broken it out into the components of the data structure below.</p><h3>Stack Definition</h3><p>We import deque from the collections library and define our stack as a new deque object.</p><pre>from collections import deque</pre><pre>class Stack:<br>    def __init__(self):<br>        self.container = deque()</pre><h3>Push Function</h3><p>Within our Stack class we define a push() function that takes an element to be appended onto the stack.</p><pre>class Stack:<br>    def push(self,val):<br>        self.container.append(val)</pre><h3>Pop Function</h3><p>Within our Stack class we define a pop() function that removes the top element from the stack.</p><pre>class Stack:<br>    def pop(self):<br>        return self.container.pop()</pre><h3>Peek Function</h3><p>Within our Stack class we define a peek() function that returns the top element on the stack.</p><pre>class Stack:<br>    def peek(self):<br>        return self.container</pre><h3>Empty Checking Function</h3><p>Within our Stack class we define an is_empty() function that checks if the length of the stack is zero and returns a boolean (True/False).</p><pre>class Stack:<br>    def is_empty(self):<br>        return len(self.container)==0</pre><h3>Size Function</h3><p>Within our Stack class we define a size() function that returns the length of the stack.</p><pre>class Stack:<br>    def size(self):<br>        return len(self.container)</pre><h2>Even More</h2><ul><li>Queues are often considered the counterpart to the Stack data structure because they have a first in, first out (FIFO) property instead of stacks’ LIFO property. I will cover the queue data structure in a future post.</li><li>Remember that stacks are often implemented on top of a singly linked list data structure which manages the elements of a stack as nodes in a linked list. Linked lists are useful for stacks because only the end element needs to be accessed for both data structures. Check out my blog post about <a href="/notes/coding-data-structures-linked-lists">Linked lists here</a>.</li></ul><p>That’s stacks!
